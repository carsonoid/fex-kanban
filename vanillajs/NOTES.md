## Thoughts

- I miss types. But pure javascript means _pure javascript_ for this exercise.
- My editor is still actually pretty good about infering types and having good intellisense with even pure javascript.
- In some ways I feel like a beginner again. It's surprising how much you take for granted with even a basic framework.
- It's also freeing to see and write 100% of every aspect of the site.

  Without that, you have to start building a UI from the "list" page but there is no good example of content.
- I know enough about javascript history to be glad I have `fetch` native and I'm not manually writing AJAX code.

## Conclusion

Yeah, I can see why we very quickly wanted ways to ease the pain here. It is really nice to see that everything can be done
manually. It's also great to have a fully functional, dynamic site where 100% of the code exists in 3 files. Even having 3 was a design choice, inline css and javascript could have had it all in one file.

I've got a basic board list working with an add modal and cheesy refresh (clear and reload). I won't be winning any style awards but
it has been a nice return to basics experience.
